Designing for Web Accessibility

By Gicelle, Dylan and Raghav


This webpage provides recommendations for user interface design (UI) and visual design more accessible to people with disabilities.


  1. Contrast with Foreground and Background

    • The colour of text in the foreground must contrast the background colour for information to be more easily conveyed. This includes images, buttons, and any other elements.
    • The correct terminology for this is “luminance contrast ratio”, or “contrast ratio” for short.
    • However, this does not apply to logos or and incidental text such as text in images.

      • Example:

      Insufficient

      Some people cannot read text if there is not sufficient contrast between the text and background. For others, bright colors (high luminance) are not readable; they need low luminance.


      Sufficient

      Some people cannot read text if there is not sufficient contrast between the text and background. For others, bright colors (high luminance) are not readable; they need low luminance.


  2. Do NOT use colours alone!

    • Colours are useful for contrasting, which can make information more easily conveyed to the user. However, some users may be colour-blind, making colours difficult to interpret.
    • Thus, using other identification that is not focused on colour such as symbols and numbers.

  3. Simple Interactive UI

    • When providing interactive elements (e.g. buttons, links), make sure they are easy to identify.
    • This can be achieved by having:
      1. Mouse hover style
      2. Keyboard focus style
      3. Touch or click style

    Example:

  4. Simple Navigation

    • Ensure that navigation (e.g. links to another page/website) has consistent naming, styling and positioning.
    • Do NOT only have one way of navigation, and implement different types (i.e. site search or a site map)
    • Use orientation cues (e.g. breadcrumbs - like a file directory) to help user experience and help them understand where they are in a website.

  5. Clear Labels

    • Ensure that all fields have a descriptive label adjacent to the field.
    • Position labels to the left or above the field (except for checkboxes and radio buttons - typically to the right)
    • Avoid having too much space between labels and fields.

  6. Identifiable Interaction Feedback

    • This includes confirming form submission, alerting the user when something goes wrong, or notifying the user of changes on the page. This should be easy to identify.
    • Important feedback should be easily noticeable and identifiable, using contrast (colours and symbols).

  7. Headings and Spacing

    • Use whitespace to make the content more clear. Style classes can be used to group content, add whitespace, and make it easier to scan and understand.

    Example:

  8. Different Viewport Sizes

    • Ensure the webpage can adapt to different screen sizes. Consider positioning, text-size, etc.
    • Ensure that text size and line width are set to maximize readability and legibility.
    • Large screens can use multiple columns, visible navigation options, and visible secondary information.
    • Smaller screen can use a single column for content and icons (for navigation options and secondary information).

  9. Images and Media

    • Although this is good for visuals and makes the UI more aesthetic, provide alternatives:
      1. Transcript of audio
      2. Audio description of videos
      3. Text with buttons/icons
      4. Captions and descriptions for tables or complex images

  10. Provide Controls for Immediate Content

  • Provide visible controls to allow users to stop any animations or auto-playing sound.
