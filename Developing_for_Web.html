Web Accessibility Initiative | Developing for web accessibility

Preface

This page introduces some basic considerations to help you get started developing web content that is more accessible to people with disabilities.
These tips are good practice to help you meet Web Content Accessibility Guidelines (WCAG) requirements.

Key points

  • Web accessibility is needed to help those with disabilities
  • There is the Web content accessibility guidelines (WCAG) which developers should follow
  • Developers should have the order of the code be the same as the order in which the page is read.
  • Code should be in a way that makes it fit on a variety of screen sizes (phone, laptop, etc)
  • Use tabindex = 0 in the CSS file to make important buttons keyboard accessible
  • Use as little CAPTCHA as possible, as it can cause problems for people with disabilities
  • Identify and use user's native language to make user experience more comfortable

Development Standards

  • Include alternative text for images. For example, have an alt="..." on the images so they know what it is
  • Identify page language by including such code in your HTML: <html = lang="en">
  • Have each HTML element have its own class so that you can edit it easier with CSS
  • Order the CSS and HTML in the order it is displayed as the page is meant to be read
  • Allowing for users to understand your website is crucial. When you add in features that aren't as widely used,
    they can be confusing for some people. Providing insight to what these features do can allow for an easier user
    experience when interacting with your website
  • Ensure that all interactive elements are keyboard accessible. Make it so that when you click a specific button,
    an action happen which you can also do with your mouse.
  • For CAPTCHA: Provide more than 2 ways to solve it, and do not require CAPTCHAs for authorised users.
