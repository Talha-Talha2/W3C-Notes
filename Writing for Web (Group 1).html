Writing for Web Accessibility

Introduction:

Writing for Web Accessibility refers to implementing web content that is more accessible to those with disabilities.

Informative, Unique Page Titles:

Make sure the titles for web pages are relevant and succinctly summarise the web page contents in a few words. Examples of bad and good page titles could be:

  • Webpage ❌
  • A webpage summarising the information from the website “Writing for Web Accessibility”, which was written by the World Wide Web Consortium. This page introduces some basic considerations to help you get started writing web content that is more accessible to people with disabilities. These tips are good practice to help you meet Web Content Accessibility Guidelines (WCAG) requirements. Follow the links to the related WCAG requirements, detailed background in the “Understanding” document, guidance from Tutorials, user stories, and more. ❌
  • Information on Web Accessibility ✅

Use headings to convey meaning and structure:

  • Use heading and subheadings to clearly describe sections.
  • Provide a descriptive outline of the content.
  • Use headings to divide pieces of information on a webpage.

Meaningful Link names:

  • When making links, make sure they are meaningful and convey what they link to. E.g.:
  • More information of Information ✅
  • Click here ❌

Meaningful text alternatives for images

When writing alternative text for images (the text that appears instead if the image can’t or doesn’t load), make sure that text is descriptive and describes what the image should look like.

For this image, the alternative text shouldn’t or should look like:

  • Skeleton ❌
  • An image of a pixelised skeleton, with a single blue eye, wearing a blue hoodie, black pants, and pink slippers. Sans undertale ✅

Create Transcripts and Captions for Multimedia:

  • Transcripts should be provided for audio-only content
  • Captions should be provided for audio visual content
  • These can provide important information and can improve accessibility for individuals who may be deaf for example

Provide Clear Instructions:

  • Ensure that instructions, guidance, and error messages are clear, easy to understand, and avoid unnecessarily technical language.
  • Describe the input requirements, such as date formats

Keep Content Clear and Concise:

This involves using using simple language and formatting to make the website easier to understand and follow for the reader. Features that should be used include:

  • Simple language and formatting makes the website easier to understand and follow
  • Short, clear sentences and paragraphs should be used
  • Unnecessary complex words and phrasings should be avoided
  • Acronyms should be shown in expanded form first
  • A glossary of terms may help the reader to understand the content better
  • Lists can help to simplify information
  • Images and diagrams can provide additional information

These features improve the readability of the webpage, making it more accessibility to a wide variety of readers.
